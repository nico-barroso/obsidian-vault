   **Docker** es una plataforma de **contenedores** que permite **empaquetar una aplicación con todas sus dependencias** y ejecutarla de forma **aislada, reproducible y portable**, independientemente del sistema donde se despliegue.

En lugar de instalar software directamente en el sistema operativo, Docker ejecuta aplicaciones dentro de **contenedores**, que son instancias ligeras creadas a partir de **imágenes**.


#  ¿Por qué Docker?

Docker se utiliza principalmente para:
1. Garantizar que una aplicación funcione igual en **desarrollo, pruebas y producción**.
2. Evitar problemas de "en mi máquina funciona"
3. simplificar despliegues y entornos complejos
4. Facilita el trabajo con microservicios
5. Aisla aplicaciones sin el coste de máquinas virtuales complejas

# ¿Cómo funciona?

De forma simplificada, Docker se basa en:
1. **Imágenes**: plantillas inmutables de una aplicación
2. Contenedores: ejecuciones vivas de una imagen
3. Docker Engine: el motor que gestiona imagenes y contenedores
